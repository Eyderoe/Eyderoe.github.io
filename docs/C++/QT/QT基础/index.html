Qt 基础

基于 C++ 的 GUI 框架，参考图书《Qt 5.9 C++ 开发指南》

01 CLion × Qt 环境配置​

基础​

  1. 安装图书对应 Qt 版本 Qt 5.9

  2. 安装时勾选 MinGW 32

  3. 打开 Qt Creator 并构建运行一个示例，来检测安装是否成功

CLion 配置​

新版本的 CLion 对 Qt 的默认支持已经非常好了 (2025.1)

  1. 将 MinGW 目录加入环境变量，xxx\Qt5.9.1\5.9.1\mingw53_32\bin
  2. 在 CLion 中添加 Qt 工具链，xxx\Qt5.9.1\Tools\mingw530_32

创建项目​

CLion​

  1. 创建项目时 Qt CMake 前缀路径，xxx\Qt5.9.1\5.9.1\mingw53_32
  2. 项目创建后在 CMake 使用 Qt 工具链新建 DebugQt
  3. 创建带 ui 文件的文件时 (cpp hpp ui) 注意文件基大小写，不一致可能导致无法通过 creator 生成代码

若生成 .ui 文件后提示找不到 ui_widget.h ，需要在 CMkaeLists.txt 中添加 include_directories(xxx/cmake-build-debug-mingwqt/xxx_autogen/include)

Qt Creator​

Qt Creator 的 go to slot 功能非常好用，但通过外部打开 .ui 文件后不能使用该功能

  1. 回到 Qt Creator，打开项目
  2. 工具链选择 Qt 默认，因为编译只会在 CLion 中进行，所以不会产生额外的文件夹 (不知道为什么选择 CLion 的 cmake 目录看不到项目中的文件)

02 文件机制​

Qt 中任何窗体或界面组件都是用类封装的。有点像 STM32 的 HAL 库，只有某些东西配置后再用 Qt Creator 生成，才能在代码里面调用，不过不是宏定义而是真正的类

.pro 为 Qt 过时的项目管理文件，现代 Qt 通常使用 cmake 而不是 qmake

这里用 QWidget 作为基类生成

  • QWindow 基础类 最底层抽象
  • QWidget 核心类 提供了控件和布局
  • QDialog 特殊窗口 继承自 Widget

widget.ui​

.ui 文件是 xml 格式，描述了界面

Qt Creator 打开 ui 文件后右下角描述了继承关系，上面是父类，下面是子类

ui_widget.h​

是根据组件、信号和信号槽自动生成的文件，里面定义了 Ui_widget 并派生了 widget

class Ui_widget

namespace Ui
{
class widget : public Ui_widget {};
} // namespace Ui

widget.hpp/.cpp​

创建 ui 时自动生成的文件 (.cpp .hpp .ui)，其中定义了 widget，注意与 Ui 命名空间下的 widget 不是一个类。Ui::widget 是纯 UI 相关的类，widget 是主窗口类

namespace Ui
{
class widget;
}

class widget : public QWidget {
        Q_OBJECT // 信号槽机制宏
    public:
        explicit widget (QWidget *parent = nullptr);
        ~widget () override;
    private:
        Ui::widget *ui; // 指向可视化界面的指针
};

widget::widget (QWidget *parent) : QWidget(parent), ui(new Ui::widget) {
    ui->setupUi(this); // 实现了组件中设置、信号和槽的关联
}

05 混合编程 (项目1)​

虽然代码能实现更多特性，但能用界面编程建议用界面编程

.qrc，qt 资源文件比如一些小图标。在 Qt Creator 中生成后，需要回到 CLion 在 CMkaeLists.txt 中的添加可执行文件处添加对应 .qrc 文件

add_executable(smpl05 res.qrc)

软件图标：根目录新建 app.rc 和 app.ico ，并在 rc 文件中编辑，同样也需要添加至CMkaeLists.txt

IDI_ICON1 ICON DISCARDABLE "20.ico" // app.rc

add_executable(smpl05 app.rc) // CMakeLists.txt

对于涉及多态的绑定，建议使用 qOverload

QObject::connect(spinBox,SIGNAL(valueChanged(int)), this,SLOT(handleFontSize_valueChanged)); // 旧式
QObject::connect(spinBox, qOverload<int>(QSpinBox::valueChanged), this, &mainwindow::handleFontSize_valueChanged); // 新式

07 Qt 核心 - 元对象系统​

元系统：信号槽、对象树等核心机制

元对象编译器 (MOC)：预处理，将 Q_OBJECT 宏的 Qt 程序转换为标准 C++

// 如果不是在头文件中写的话，MOC会处理这一块东西
class QMyWidget : public QWidget {
        Q_OBJECT
        Q_CLASSINFO("autor", "xxx") // 第0个属性
        Q_PROPERTY(bool enabled READ isEnabled WRITE setEnabled)
    public:
        explicit QMyWidget (QWidget *parent = nullptr) {};
        ~QMyWidget () {};
};

int main (int argc, char *argv[]) {
    QApplication a(argc, argv); // Qt其他东西出现前必须有这玩意

    QObject *obj = new QPushButton; // QObject 是所有元对象系统基类
    std::cout << obj->metaObject()->className() << std::endl; // 返回类名
    std::cout << obj->inherits("QObject") << std::endl; // 返回是否是子类

    QObject *obj2 = new QMyWidget;
    QWidget *widget = qobject_cast<QWidget*>(obj2);
    QMyWidget *myWidget = qobject_cast<QMyWidget*>(obj2);
    QPushButton *button = qobject_cast<QPushButton*>(obj2); // 不会报错, null
}

08 Qt 核心 - 属性、信号槽​

Q_PROPERTY 宏定义了一个返回类型为 type，名称为 name 的属性

Q_CLASSINFO 定义了一些其他奇奇怪怪的信息

int main (int argc, char *argv[]) {
    QObject *obj3 = new QMyWidget;
    obj3->setProperty("flat", true); // 没有提取声明, 动态属性
    bool flat = obj3->property("flat").toBool(); // 这里是返回的 QVariant, 需要通过方法才能转换至对应类型

    QMyWidget *myWidget2 = new QMyWidget;
    std::cout << myWidget2->metaObject()->classInfo(0).name();
    std::cout << myWidget2->metaObject()->classInfo(0).value();
}

信号与槽的连接

  • Auto 自动 (缺省)
  • Direct 同一线程 发送立刻执行
  • Queued 不同线程 排队 ? 异步
  • Blocking 柱塞

SP1 整理回顾​

大致回顾 Qt 并巩固 Qt 核心内容

  • 界面: 可以通过 Creator 可视化编辑也能通过代码直接编写
  • QObject: 绝大多数的基类，父对象析构时会自动析构所有子对象。加上 Q_OBJECT 宏可使用 Qt 元对象系统
  • 信号槽: 信号和槽，有点像发布订阅，通过 connect 关联。信号既可以通过 Qreator 可视化编辑也能通过代码直接编写
  • 事件循环: 调用 exec() 进入事件循环，不断从事件队列中取出事件并分发给目标对象

10 Qt 对标准库的补充​

全局定义​

QtGlobal 包含了基本数据类型、函数、宏，但一般其他文件会包含所以不需要自己显式包含。基本数据类型现代 C++ 也实现的差不多了 qint64 <=> int64_t

容器​

对 Qt 的兼容性要好一些，此外 QString 对 unicode 的支持较好，方法也非常便捷

迭代器分为 Java、STL 类型。Java 风格在很多情况下比较简单，但理念与 STL 有些差别

QVector<int> container;
// Java
QMutableVectorIterator<int> j(container);
while (j.hasNext()) {
    if (const int value{j.next()}; value == 1)
        j.remove();
}
// STL
for (auto it = container.begin(); it != container.end();) {
    if (*it == 1)
        it = container.erase(it);
    else
        ++it;
}
