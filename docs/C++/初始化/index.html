变量初始化

初始化方式以及构造函数

初始化方式​

C++ 提供了多种初始化方式 {}, =, ()

构造初始化 =​

从 C 继承的初始化方式，隐式转换

T object = value;

首先会通过 value 创建一个临时对象，并通过临时对象初始化 object。但现代编译器均会直接调用构造函数来避免额外拷贝开销，C++17 后该优化必须被实现

直接初始化 {}​

C++11 引入的统一初始化语法

T object{arg1, arg2, ...};

禁止了窄化转换，会直接导致编译错误

其他不正确的使用，也会导致出现问题。初始化列表的构造顺序：

  1. 尝试形参为初始化列表的构造（最优先匹配）
  2. 其他构造函数
  3. 聚合初始化，依次构造成员。但要求没有声明构造函数

聚合初始化可以缺省值。缺省值使用值初始化，内置类型初始化为 0 等，类会调用默认构造函数

值初始化 ()​

C++98 风格

T object(arg1, arg2);

与直接初始化不同，允许窄化操作

构造函数​

C++ 提供了四大拷贝移动函数：拷贝构造、拷贝赋值、移动构造、移动赋值。假如声明了以上任一一个或析构函数，编译器会自动生成移动和构造构造函数

class vector {
    public:
        // 默认构造
        vector () {
            std::cout << "default constructor" << std::endl;
        }
        // 拷贝构造
        vector (const vector &other) {
            std::cout << "copy constructor" << std::endl;
            size = other.size;
            data = new int[size];
            std::copy(other.data, other.data + size, data);
        }
        // 拷贝赋值
        vector& operator= (const vector &other) {
            std::cout << "copy assignment" << std::endl;
            if (this != &other) { // 自赋值情况
                delete[] data;
                size = other.size;
                data = new int[size];
                std::copy(other.data, other.data + size, data);
            }
            return *this;
        }
        // 移动构造
        vector (vector &&other) noexcept {
            std::cout << "move constructor" << std::endl;
            // 复制
            data = other.data;
            size = other.size;
            // 设置为可析构
            other.data = nullptr;
            other.size = 0;
        }
        // 移动赋值
        vector& operator= (vector &&other) noexcept {
            std::cout << "move assignment" << std::endl;
            if (this != &other) {
                // 自赋值情况
                delete[] data;
                // 复制
                data = other.data;
                size = other.size;
                // 设置为可析构
                other.data = nullptr;
                other.size = 0;
            }
            return *this;
        }
    private:
        int *data{};
        size_t size{};
};

返回值优化​

编译器一般会启用 RVO NRVO 优化，可以通过 CMake 选项禁止。C++17 及以后强制要求优化

target_compile_options(clionTest PRIVATE -fno-elide-constructors)

有以下代码

vector make_vector () {
    return vector();
}
vector v(make_vector());

在开启优化的情况下，仅会调用一次默认构造

在关闭优化的情况下，如果声明了移动构造，会调用一次默认构造和两次移动构造；如果仅声明了拷贝构造，则会调用一次默认构造和两次拷贝构造
