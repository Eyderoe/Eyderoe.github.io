移动语义

现代 C++ 中非常重要的特性，解决了不必要的拷贝

变量类型和值类别​

理解这一坨的一个关键是区别 变量 和 表达式

变量​

每个变量在编译时期都会确定其变量类型，并在整个生命周期中不会改变

表达式​

每个表达式有两个属性：类型和值类别

  • 类型：表达式计算后得到的本身
  • 值类别：左值、将亡值、纯右值（后面会细说）

左右值​

左值：指向特定内存具有特定名称的值

右值：临时的表达式或值

int x{};
x++; // 生成临时 然后递增x 并返回临时
++x; // 递增 返回本身引用
int *p = &(++x); // 左值 故可以取地址
int *q = &(x++); // 警告 不能将运算符 '&' 应用于右值

左右值引用​

引用即别名，一旦绑定不能修改

左值引用​

非常量引用只能引用左值，常量引用除了左值还能引用右值

int x{};
int &y = x;
const int &z = 0; // 延长了生命周期

右值引用​

C++ 中很多操作会返回很多右值，比如函数按值返回、表达式中间值

右值引用只能绑定右值（非通用引用时）

int x{};
int &&y = 0; // 纯右值 prvalue 延长了生命周期
int &&z = static_cast<int&&>(x); // 需要 static_cast 进行转换 将亡值 xvalue

左值不能被隐式移动，但能通过 move 或者 static_cast 显式的将左值转换为右值。将亡值虽然没有名称但拥有地址，与纯右值区分

通用引用​

在类型确定的情况下，很正常

void func (int &x); // 接受左值
void func (int &&x); // 接受右值
int x{};
int &&y{};
func(std::move(x)); // 调用 int &&x
func(y); // 调用 int &x

但当类型变为模板时，情况就有所不同了。不是普通的右值引用，而是通用引用（未显式实例化模板时）

template <typename T>
void func (T &&x);
int x{};
func(x); // 左值 T推导为 int&, int& && -> int&
func(0); // 右值 T推导为 int, int -> int&&

实参为左值会被推断为带引用类型，实参为右值会被推断为基础类型

左值流程：

  1. 尝试 T = int，形参类型为 int &&
  2. int && 不能绑定到左值 a
  3. 尝试 T = int&，形参类型为 int& &&
  4. 折叠为 int&，匹配左值

右值流程：

  1. 尝试 T = int，形参类型为 int &&
  2. 匹配右值

涉及到 C++ 中另一个重要特性，引用折叠规则

  • 只要其中一个是左值引用，结果就是左值引用
  • 只有两个都是右值引用，结果才是右值引用
using lref = int&;
using rref = int&&;

int x{};
lref &a = x; // int& & → int&
lref &&b = x; // int& && → int&  
rref &c = x; // int&& & → int&
rref &&d = 10; // int&& && → int&&

此时可以通过 forward 关键字实现完美转发 (保留值类型)

template <typename T>
void wrapper (T &&x) { // x 是左值
    func(std::forward<T>(x)); // 保留原来的值类别
}
