X-Plane SDK

SDK 包允许编写插件来控制 X-Plane，官网的文档也非常详细

预先准备​

仍然推荐使用 cmake 进行管理，虽然官网提供的示例代码是与 IDE 关联的。下面是经过验证可直接使用的工具链：

  • Win：MinGW、MSVC
  • Mac：Apple Clang

这里给出一个完整的 CMake 示例

cmake_minimum_required(VERSION 3.20)
project(XPTest)

set(CMAKE_CXX_STANDARD 20)
if (APPLE)
    add_definitions(-DAPL=1 -DIBM=0 -DLIN=0)
    set(CMAKE_OSX_ARCHITECTURES "x86_64;arm64") # Intel和Silicon
    set(SDK_ROOT "/Users/eyderoe/CppLib/SDK")
elseif (WIN32)
    add_definitions(-DAPL=0 -DIBM=1 -DLIN=0)
    set(CMAKE_MSVC_RUNTIME_LIBRARY "MultiThreadedDLL")
    add_definitions(-D_CRT_SECURE_NO_WARNINGS)
    set(SDK_ROOT "E:\\cxx_project\\_lib\\SDK")
endif ()

include_directories(
        ${SDK_ROOT}/CHeaders/XPLM
        ${SDK_ROOT}/CHeaders/Widgets
)

if (APPLE)
    add_definitions(-DAPL=1 -DIBM=0 -DLIN=0)
elseif (WIN32)
    add_definitions(-DAPL=0 -DIBM=1 -DLIN=0)
endif ()
add_definitions(
        -DXPLM303 # 11.5
        -DXPLM301
        -DXPLM300
        -DXPLM210 # 10.2
        -DXPLM200
)

add_library(HelloWorld SHARED
        "hello world.cpp"
)

set_target_properties(HelloWorld PROPERTIES PREFIX "" SUFFIX ".xpl")

if (WIN32)
    target_link_libraries(HelloWorld "${SDK_ROOT}/Libraries/Win/XPLM_64.lib")
elseif (APPLE)
    target_link_libraries(HelloWorld "-undefined dynamic_lookup")
endif ()

概要​

SDK 与宏​

XPlane SDK 的对 XPlane 版本的兼容仅根据 XPLM 宏决定，过时的 SDK 可能不支持最新的平台（比如 XP12 后才正式支持 Apple Silicon），所以无论目标版本如何，都应该使用最新的 SDK

  • XPLM400：XP12.04 及以后
  • XPLM303：XP11.50 及以后

虽然 XPLM 宏显然呈现一种大小关系，但实际没有，若要支持低版本需要写对应低版本的宏

程序入口​

分为两组：

  • XPluginStart、 XPluginStop：构造函数和析构函数，这一组是必须的
PLUGIN_API int XPluginStart (char *outName, char *outSig, char *outDesc)
PLUGIN_API void XPluginStop (void)
  • XPluginDisable、XPluginEnable：插件被禁用和启用时的动作，非必须但不声明 log 里面会警告找不到
PLUGIN_API void XPluginDisable (void)
PLUGIN_API int XPluginEnable (void)
PLUGIN_API void XPluginReceiveMessage (XPLMPluginID inFrom, int inMsg, void *inParam)

没有找到这一块在 sdk 文档里的说明，很奇怪，需要返回 int 默认 1 就行

XPLMDataAccess​

基本​

主要通过 Dataref 对游戏内数据进行读写，这里除了对官方数据的读写还有自定义数据等内容。在整个模拟器的运行周期中 Dataref 不会被销毁，即使是提供该数据的插件被禁用或销毁，所以可以通过索引来唯一标识

typedef void * XPLMDataRef;
XPLM_API XPLMDataRef XPLMFindDataRef (const char *inDataRefName);

定义了一个句柄。需要的 dataref 名称和类型可以在第三方网站查询，并通过 XPLMFindDataRef 来获取。每次 XPlane 启动时都会重新注册，所以需要通过查找来实现，而不是硬编码 ID

读写​

对于每种支持的数据类型，均有一个读函数和写函数。因为这是 C 库，所以也没有 C++ 的重载，int float double byte 都有对应函数

// 单个数据
XPLM_API int XPLMGetDatai (XPLMDataRef inDataRef);
XPLM_API void XPLMSetDatai (XPLMDataRef inDataRef, int inValue);
// 序列数据
XPLM_API int XPLMGetDatavi (XPLMDataRef inDataRef, int *outValues, int inOffset, int inMax);
XPLM_API void XPLMSetDatavi (XPLMDataRef inDataRef, int *inValues, int inoffset, int inCount);

XPLMProcessing​

通过注册回调来完成周期性任务，频率不会超过物理计算帧率，以及禁止在回调中进行绘制。按时间调度时可能会略微晚一点，因为真正的调度为超时后的第一次尝试调度

基本回调​

这是标准的回调函数定义，返回值可以更新下一次回调时间

typedef float (*XPLMFlightLoop_f) (float inElapsedSinceLastCall, float inElapsedTimeSinceLastFlightLoop, int inCounter, void *inRefcon);

并通过 XPLMRegisterFlightLoopCallback 和 XPLMUnregisterFlightLoopCallback 进行注册和注销

XPLM_API void XPLMRegisterFlightLoopCallback (XPLMFlightLoop_f inFlightLoop, float inInterval, void *inRefcon);
  • inFlightLoop：回调函数
  • inInterval：下一次调用时间，-1 为下一帧即调用
  • inRefcon：自定义的一些东西，回调函数中会原封不动的接收

高级回调​

XPLMRegisterFlightLoopCallback 只能创建飞行循环前，更高级的操作允许自定义调度，包括创建飞行循环后的调度和自定义调度器

同样的，可以通过句柄追踪一个回调

typedef void *XPLMFlightLoopID;
