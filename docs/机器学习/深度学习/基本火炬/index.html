线性模型​

训练集、测试集、验证集

  • 训练集：用于训练模型，学习参数的数据
  • 测试集：拿来测试模型的数据
  • 验证集：训练集分出来的一部分，用于模型评估（调整参数）防止过拟合

对于下面的数据集，有对应线性模型 y^=x∗ω+b\hat{y} = x * \omega + by^​=x∗ω+b，这里 y^\hat{y}y^​ 指模型预测 ω\omegaω 称为权重

(x,y)
(1,2)(2,4)(3,6)(4,8)

肉眼很容易能看出，这个函数的表达式 y=2xy=2xy=2x。但计算机只能进行猜测，比如 y^=x\hat{y}=xy^​=x 猜测以后通过评估才能改进。深度学习里有损失函数，目标就是求损失函数最小，比如用均方误差（Mean Square Error）作为损失函数

loss=(y^−y)2=(x∗ω−y)2loss = (\hat{y} - y)^2 = (x * \omega - y)^2loss=(y^​−y)2=(x∗ω−y)2

损失函数是针对样本的，针对数据集有：cost=1N∑n=1N(y^n−yn)2cost = \frac{1}{N} \sum_{n=1}^{N} (\hat{y}_n - y_n)^2cost=N1​∑n=1N​(y^​n​−yn​)2，优化问题可表示为 ω∗=arg⁡min⁡ωcost(ω)\omega^* = \arg\min_{\omega} cost(\omega)ω∗=argminω​cost(ω)

梯度下降​

上面其实只进行了评估，但并没有进行改进。并且随着参数的增多，上面穷举的效率太低了，且可能是局部最优

对于这样的一个模型 y^=x∗ω+b\hat{y} = x * \omega + by^​=x∗ω+b，沿着梯度下降的方法总能找到最小值。更新方法为 ω=ω−α∂cost∂ω\omega = \omega - \alpha \frac{\partial cost}{\partial \omega}ω=ω−α∂ω∂cost​ 其中 α\alphaα 是学习率。虽然在其他复杂模型中不一定是全局最优，梯度下降能高效且稳定的找到局部最优（经过长期实践发现局部最优解其实并不多），最大的问题是陷入鞍点

训练一般都会收敛，发散可能是因为学习率太大导致训练失败

梯度下降有改进，比如随机梯度下降 ω=ω−α∂loss∂ω\omega = \omega - \alpha \frac{\partial loss}{\partial \omega}ω=ω−α∂ω∂loss​，可以解决卡在鞍点的问题。由于随机梯度是通过每个样本进行更新，难以进行并行计算，可以通过把数据集给切成多份（Batch）来训练（小批量梯度下降）

反向传播​

这里是最基本的网络 y^=x∗ω+b\hat{y} = x * \omega + by^​=x∗ω+b

训练时是对权重 ω\omegaω 进行更新，简单的模型能直接求梯度。复杂的网络中显然不能直接求梯度，因为维度会变。但可以看成向量，权重会是一个大矩阵，此时还是很难

这里输入是 5∗15*15∗1 中间第一层是 6∗16*16∗1 那权重就会是 6∗56*56∗5 矩阵，即 h=ω∗xh=\omega*xh=ω∗x

再把复杂的矩阵简单化，对于 y^=W2(W1⋅X+b1)+b2\hat{y} = W_2(W_1 \cdot X + b_1) + b_2y^​=W2​(W1​⋅X+b1​)+b2​ 有下图，其中 bbb 为偏置。在经过研究后，发现假如每一层都是这样线性，其实都可以写成一层的形式。所以每次都要加一个非线性函数作变换

想一下复合函数求导 f(g(x))f(g(x))f(g(x))，最终结果是从里面的 xxx 一步步往外求（前馈），但算导数是一步步从外到内链式求导（反向）。这里是每个神经元进行传递，把大矩阵拆成了小矩阵

Tensor 是存储数据的基本单位 Data(权重)、Grad(导数)

import torch

forwardFunc = lambda x: x * w  # 前向计算预测值
lossFunc = lambda x, y: (forwardFunc(x) - y) ** 2  # 计算损失

x_data = [1, 2, 3]  # 数据集
y_data = [2, 4, 6]
w: torch.Tensor = torch.Tensor([1])  # 权重
w.requires_grad = True

l = 0
for epoch in range(10):  # 训练轮次
    for x, y in zip(x_data, y_data):
        l: torch.Tensor = lossFunc(x, y)
        l.backward()
        print("grad:", x, y, w.grad.item())
        w.data -= 0.01 * w.grad.data
        w.grad.data.zero_()  # 默认行为是累计梯度
    print("epoch:", epoch, l.item(), end="\n-------------\n")
print("4 predict: {}".format(forwardFunc(4).item()))

线性回归​

比上一章更为普遍的写法，一般可以分成这样几步

  1. 准备数据集
  2. 设计模型
  3. 构造损失和优化器
  4. 训练

一般把一个权重+偏置称为线性单元

下面这部分代码的写法就跟上面不太一样了，比如上面变成二次函数就是多几个 Tensor 就能解决的问题。下面这个代码更为一般，扩充的特征 x，而不是针对权重 w 和偏置 b

import torch
 
# 数据集
x_data = torch.Tensor([[1], [2], [3]])
y_data = torch.Tensor([[2], [4], [6]])
# 设计模型
class Linear(torch.nn.Module):
    def __init__(self):
        super().__init__()
        self.linear = torch.nn.Linear(1, 1)  # 二次函数的话就是(2,1) 两个特征

    def forward(self, x):
        # x = torch.cat([x, x ** 2], dim=1)  # 二次函数要扩充特征 x -> [x,x^2]
        y_pred = self.linear(x)
        return y_pred
    
model = Linear()
# 损失和优化器
criterion = torch.nn.MSELoss(reduction="sum")
optimizer = torch.optim.SGD(model.parameters(), lr=0.01)
#训练
for epoch in range(200):
    y_preds = model(x_data)
    loss = criterion(y_preds, y_data)
    optimizer.zero_grad()
    loss.backward()
    optimizer.step()
print("w={:.2f}, b={:.2f}".format(model.linear.weight.item(), model.linear.bias.item()))
print("x=4, y_pred={:.2f}".format(model(torch.Tensor([[4]])).data.item()))

虽然叫回归，但其实做的分类任务。经典的分类任务有：

  • CIFAR-10：小尺寸彩色图片

先来一点逻辑函数：σ(x)=11+e−x\sigma(x) = \frac{1}{1 + e^{-x}}σ(x)=1+e−x1​，和 tanhtanhtanh 等一大堆函数合称饱和函数

对应的模型为 y^=σ(x∗ω+b)\hat{y} = \sigma(x * \omega + b)y^​=σ(x∗ω+b)，损失函数也变为 loss=−(ylog⁡y^+(1−y)log⁡(1−y^))loss = -(y \log \hat{y} + (1 - y) \log(1 - \hat{y}))loss=−(ylogy^​+(1−y)log(1−y^​))，这里 y^\hat{y}y^​ 表示预测类别的概率，同样的，针对数据集有平均交叉熵损失 J(w,b)=−1m∑i=1m[y(i)log⁡(y^(i))+(1−y(i))log⁡(1−y^(i))]J(w, b) = -\frac{1}{m} \sum_{i=1}^{m} [y^{(i)} \log(\hat{y}^{(i)}) + (1-y^{(i)}) \log(1-\hat{y}^{(i)})]J(w,b)=−m1​∑i=1m​[y(i)log(y^​(i))+(1−y(i))log(1−y^​(i))]

多分类​

单分类很简单，就两个类别。单多分类会产生一堆类别，不能保证合为 1，故引入 Softmax 使和为 1

P(y=i)=ezi∑j=0K−1ezj,i∈{0,...,K−1}P(y = i) = \frac{e^{z_i}}{\sum_{j=0}^{K-1} e^{z_j}}, i \in \{0, ..., K-1\}P(y=i)=∑j=0K−1​ezj​ezi​​,i∈{0,...,K−1}

修改输出后，会发现均方差损失也不再适合多分类任务，改为交叉熵损失：Loss(Y^,Y)=−Ylog⁡Y^Loss(\hat{Y}, Y) = -Y \log \hat{Y}Loss(Y^,Y)=−YlogY^。不过在 PyTorch 中交叉熵损失模块已包含 Softmax

卷积网络​

上一节 FCN 基本只能达到 98% 左右就不能再提升了，因为摊平丢失了空间信息。假如进行了卷积，就可以保留空间信息，降低训练轮次的同时能进一步提升到 99%

对于多通道图像的基本概念，有：

每个卷积核对图像卷积后维度都是 1，只有用多个卷积核才能得到多个维度

由于卷积会让图片变小，可以用 padding 把图片周围进行填充

卷积是把图像的一小部分与卷积核相乘求和。还有最大池化层，把图像切分成一个一个的小块，并求每个小块中最大值

这种串行的简单卷积神经网络就很简单，也有复杂的比如 googleLeNet

其中每一个小模块有很多分支，重要能提取出特征的分支，自己权重就会提升

1x1 的卷积能显著降低通道数，因为能保证大小不变的情况下，随意更改通道数量。随着深度网络越来越深，发现模型性能不升反降，其中最主要的原因是梯度消失/爆炸。后来有聪明比发明了残差网络

如果深层网络难以学习到新特征，那么至少让它能轻松地学会恒等映射，即让输出等于输入。这样，深层网络的表现最差也不会差于浅层网络

循环网络​

能够存储信息适合处理序列数据，同一个 RNN Cell 反复参与运算（权重还是一份），显然的 这还是比较容易导致计算量爆炸，问题还包括梯度消失等

简单或复杂的

PyTorch 中实现也很方便，但需要额外注意维度信息

# 单元

cell = torch.nn.RNN()
