Java 基础

Java 启动！基于 JDK17

基础概念​

文件

Main.java 中只能放一个 public class Main，同样会将 main 方法作为程序入口（不一定需要）。其他类只能是 private

public class Main {
    public static void main(String[] args) {

    }
}

注释

// 单行
/*多行*/
/**
 * 文档
 **/

变量

和 C 差不多，这几种基本类型

int a = 0; // 整数 byte,short,...
double b = 0; // 小数 float
boolean c = false; // 布尔值
char d = ' '; // 字符 两字节

变量名称允许符号 $

输入输出

很奇怪的东西，需要声明类

import java.util.Scanner;

public class Main {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        System.out.println("Enter number: ");
        int input = sc.nextInt();
        System.out.println(input);
    }
}

运算符​

和 C 很像啊

double a = 10 / 3; // 同c整数除法
int b = (int) a; // 同c风格转换

很难绷，虽然 Java 既没有重载，String 也不是基本类型。但 String 就是能重载 +，甚至还有默认的 "" 支持，甚至能隐式转换非字符串为字符串并拼接。沟槽的

String a = "" + "";
String b = "" + 1;

自增运算符和三元也支持，可爱的捏

& 且，| 或，^ 异或，! 非。这些是非短路的

短路的还是经典 && 和 ||

判断与循环​

if 和 switch 都差不多

if (1) { // 错误的 不接受隐式转换为bool 条件必须是bool
}
switch (num) {
    case 1:
        ...
        break;
}

switch 在 JDK12 后有新特性，还支持直接赋值

switch (num) {
    case 1 -> {
        ...
    }
}

double check = switch (num) {
    case 1 -> false;
};

for while do.while 都差不多

数组​

数组默认初始化规则：基本类型是 0，引用数据类型是 null

int[] array = new int[10]; // java 样式
int brray[] = new int[10]; // c 样式

数组虽然不是类，但是特殊对象，有很多好用的特性

int[] array = new int[]{1}; // 静态初始化 地址:[I@4c873330 [数组 Iint类型 @间隔
for (int j : array) // C++ 式 foreach
    System.out.println(j);
for (int i = 0; i < array.length; i++) // C 式 fori
    System.out.println(array[i]);

Java 内存分配：栈、堆、元空间、寄存器、方法栈

int[] array = new int[]{1};
int[] array2 = array; // 指向同一块内存

方法​

回到文件架构，纯面向对象。一个文件，一个公共类，可能的一个 main 方法

public class Main {
    public static void main(String[] args) {
        
    }
}

同样支持重载，但重载不完全。对于泛型存在类型擦除，在编译时泛型会被擦除只保留原始类型，故不能重载泛型。永远达不到模板的真实

public static int sumInt(int a, int b) {
    return a + b;
}
public static double sumDouble(double a, double b) {
    return a + b;
}

面向对象​

有个小概念 JavaBean，本质上是为符合特定编码约定的类

  • 必须是公共类
  • 必须有公共无参构造函数，便于反射
  • 属性必须私有，但必须提供 Getter 和 Setter 方法
  • 最好提供序列化接口
import java.io.Serializable;/

Student student = new Student();
public class Student implements Serializable {
    private int id;

    public Student() {
    }

    public int getId() {
        return id;
    }

    public void setId(int id) {
        this.id = id;
    }
}

内存模型现在可以简单的参考 C++ 存储区：自动 动态 静态，以及编译器的常见实现：堆和栈。 但 Java 也有不同的，比如加入了方法区存储类方法，方法区在 JDK1.7 后也被元空间取代

由于 Java 中类只能 new，所以基本都放在堆内存里面

字符串​

字符串同其他语言一样， 不可变类型。不同于 C++ 没人用的 String，Java 字符串在堆上维护一个常量池，常量字符串默认进入常量池(不是 new 出来的)

直接从 byte 数组创建，好文明

byte[] bytes = new byte[]{'a', 'b', 'c'};
String str = new String(bytes);

== 操作符对一般类型比较的值，对类是比较地址。但 String 既可能在常量池，也可能不在，就导致了一小点问题

String a = "a";
String b = "a"; // a==b
String c = new String("a"); // a!=b

但 String 还能把堆中的放进常量池

String d = c.intern(); // a==d

还有 String 的可变类型版 StringBuilder，Python 的 join 方法类 StringJioner

集合​

集合只能存储引用数据类型，基本元素类型有对应引用数据类型（首字母大写）。vector map set 这些都叫集合 (

Java 中更常使用 ArrayList 而不是 Vector

ArrayList<String> list1 = new ArrayList<String>();
ArrayList<String> list2 = new ArrayList<>(); // 与 C++ 不同,声明不能简写,简写的是 new
