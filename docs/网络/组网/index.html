家庭 WiFi 组网

家庭组网攻略，默认有一定计算机网络知识

基础知识​

接口：

  • PON，只有光猫上有的东西，接光纤
  • WAN，广域网接口，但不一定是接广域，只是代表上一层级
  • LAN，局域网接口，下面的设备都是在同一网段

基础配置​

1 光猫 1 路由器

入户光纤-光猫-路由器

构成了一个最经典的配置，台式机通过网线连接路由器，其他设备通过 wifi 连接路由器

进阶配置​

1 光猫 1 主路由 n 副路由，副路由器过多的时候需要再加装一台交换机

入户光纤-光猫-主路由-副路由
入户光纤-光猫-主路由-交换机-副路由

对于现如今大多数能在网上买到的路由器，应该都支持 wifi mesh（一种 wifi 技术），即在此种情况下能自动组网完成设置。所有路由器共用一个 ssid，在一个网段下，信号弱的时候能自动切换

这种方式通过网线连接称为有线 mesh，对于主副路由器间通过 wifi 而不是网线传输数据的称为无线 mesh。注意此处无线 mesh 不同于远古时期路由器桥接技术，由于路由器桥接技术过于古老，这里不过多叙述

对于无线 mesh，阉割版 wifi 6E 少了一整个频段，但在国内厂商的努力下还是做到了 2.4 + 5.2 + 5.8 的伪三频，所以还是不用太担心速率

进阶知识​

拨号​

通过 PPPoE 协议分配到一个 ip 的过程，这个 ip 可以是公网也可以是运营商内网，反正是可以连上互联网的。在基础配置下，一般由光猫进行拨号，但进阶配置下就不同了

多路由器下一般由主路由进行拨号，光猫则为桥接模式。否则将处于光猫拨号（路由模式），主路由为动态 IP 模式，充当无线交换机的情况。这种将影响部分网络使用

网络拓扑​

对于单层楼的房子，不同房间的网口通常汇集至入户的弱电箱

对于多层楼的房子，不同楼层的房间的网口通常先汇集至每层楼单独的弱电箱的交换机，交换机再汇集到主交换机以及主路由

DHCP​

动态主机配置协议。负责分配 ip 和子网的构建，只有一台设备能进行配置（光猫或主路由），副路由需设置 AP 模式（子母机的情况一般会自动设置）

WiFi MESH​

wifi mesh 就是为了解决大空间下 wifi 的连接问题而诞生的。过去的路由器桥接技术在 mesh 出现后相形见绌，虽然都会损失一定带宽，但在延迟等方面已大幅超过。mesh 背后的支撑技术，在以下文件中被详细定义

  • 802.11 k，Radio Resource Measurement of Wireless LANs，AP 邻居信号质量测量
  • 802.11 v，Wireless Network Management，无线网络管理
  • 802.11 r，Fast Basic Service Set (BSS) Transition，快速握手

复杂情况​

单线复用​

主路由需要一进一出才能操作，但假如主路由不在弱电箱旁而房间一般只有一个网口时，有两种方案，但都基于单线复用。需要明白 VLAN 和交换机的概念

VLAN 虚拟局域网，顾名思义，能把设备划分到不同的局域网下（即使连的同一路由器）。既可以简单的通过端口划分，也能通过设备 MAC 地址进行分割

交换机，字如其名，只对数据进行交换，所以没有路由器的 WAN LAN 口而是 Trunk Access 口，理解功能后就能理解单线复用原理

  • Access，收到的数据会打上端口的 PVID（默认 VLAN）标签然后转发，发出的数据会剥离标签。因为只有交换机才识别标签，所以对终端设备的一定是 Access
  • Trunk，收到的数据带标签则检查，不带标签则打上标签；发出的数据如果 VLAN ID 和 PVID 相同则剥离标签，不同则带着标签发出

两台交换机，很简单的一种方法

光猫-交换机A [Access,VLAN2]

交换机A[Trunk,VLAN2,3]-交换机B[Trunk,VLAN2,3]
交换机A[Access,VLAN3]-副路由器[LAN] ;当然在mesh下一般WAN会退化为LAN

交换机B[Access,VLAN2]-主路由[WAN]
交换机B[Access,VLAN3]-主路由[LAN]

单臂路由，需要高级的路由器

虚拟接口，同一个网口，虚拟复用 WAN 和 LAN

IPTV​

IPTV 宽带电视，在一般的情况下，是通过 VLAN 在光猫里面通过分组分别走 IPTV 和 LAN 口。更复杂的是对于直播和点播，又分别使用了 IGMP 组播和 HTTP 单播，这都是由机顶盒完成，光猫只负责传输。所以即使组了 mesh 对直连在光猫上的机顶盒是没有影响的（VLAN41网络，VLAN45IPTV）

对于只有一根线的情况下来说就有点麻烦了

在单路由的情况下很简单，单线复用，在光猫把 IPTV 和数据都绑定到一个 LAN 口，路由器在支持 IPTV/VLAN 的情况下可以直接分一个 LAN 口作为 IPTV。简单来说就是 VLAN 各走各的

对于 mesh，参考复杂情况.单线复用.两台交换机，只不过为 IPTV 专门设置一下 VLAN45

大量物联网设备​

物联网设备一般只认 2.4g，虽然 wifi 协议自身作了大量关于兼容的工作，但还是可能导致双频合一后物联网无法连上线

关闭双频合一后，可以把 2.4g 整个划给物联网设备，并通过 VLAN 隔离整个频段，使物联网设备无法干扰其他设备
